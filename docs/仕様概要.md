# ビジネスコミュニケーション プロジェクト仕様概要

## 技術スタック
- Next.js App Router（`app` ディレクトリ）
- TypeScript + React（Client Components）
- Tailwind CSS v4（`app/globals.css` でOKLCH変数定義）
- shadcn/RadixベースのUIコンポーネント（`components/ui/*`）
- アイコン: `lucide-react`
- フォント: `next/font` の Geist/Geist Mono

## エントリ構成
- `app/layout.tsx`: メタデータとフォント設定、`<body>` でグローバルスタイル適用。
- `app/page.tsx`: `Dashboard` コンポーネントを描画。
- グローバルスタイル: `app/globals.css`（テーマ変数・reset・ベース適用）。
- セクション構成: `components/dashboard.tsx` で各セクションコンポーネントを組み立て。
  - HOT TOPICS: `components/dashboard/hot-topics.tsx`
  - 業務連絡: `components/dashboard/business-communications.tsx`
  - アナウンス: `components/dashboard/announcements.tsx`
  - Mission and values: `components/dashboard/mission-and-values.tsx`
  - 便利リンク集: `components/dashboard/link-collection.tsx`
- データ: セクションごとに `data/*.ts` へ分割（`business-communications.ts` / `announcements.ts` / `hot-topics.ts` / `mission-and-values.ts` / `link-collection.ts`）。

## 主画面 (Dashboard)
- ソース: `components/dashboard.tsx`（client component）。
- レイアウト: `main`を `lg:grid-cols-12` で二分。
  - 左8列: バナー3枚 → 「業務連絡」リスト → 「アナウンス」カードグリッド。
  - 右4列: ミッションバナー、動画風サムネイル、アコーディオン（お気に入り/人事・給与/その他/ToDo）。
- 画面上の公式ブロック名称（左上から順に、今後の呼称を統一）:
  - HOT TOPICS（トップのカード3枚領域）
  - 業務連絡
  - アナウンス
  - Mission and values
  - 便利リンク集（右カラムのアコーディオン一式を指す呼称）
- データ: コンポーネント内でモック配列を保持。
  - `businessComms`: 業務連絡一覧。type/status/title/dept/date/content を持ち、各行クリックで `Dialog` 表示。
  - `announcements`: アナウンスカード。title/date/image/content を持ち、カードクリックで `Dialog` 表示。
- UI構成要素: `Button`/`Badge`/`Card`/`Dialog`/`Accordion`/`ScrollArea` など shadcnプリミティブを Tailwind で組み合わせ。
- ヘッダー: アイコンボタン群、中央ロゴ、設定・ログアウトリンク。
- フッター: シンプルなテキストリンク群。

## スタイリング
- 基本はTailwindユーティリティ。`custom-scrollbar` クラスが参照されている（グローバルに定義想定）。
- カラーはOKLCHでトークン化（背景/前景/境界/チャート色など）。
- コーナー半径やサイドバー用トークンも用意（現状は未使用箇所あり）。

## テーマ/ユーティリティ
- `components/theme-provider.tsx`: `next-themes` ラッパー。現状未使用。
- `lib/utils.ts`: `cn` ヘルパー（`clsx` + `tailwind-merge`）。

## 今後の拡張に向けたメモ
- モックデータをAPI/DB接続に置き換える場合、`Dashboard` の state 管理とフェッチ層を切り出す。
- フィルタ/ソートボタンはUIのみ。ロジック実装時は型定義をモック配列に揃えると移行が容易。
- アコーディオンや動画エリアも固定コンテンツ。実データ化する場合は専用コンポーネントへ分割。
- テーマプロバイダが未使用のため、ダークモード対応する場合は `RootLayout` に組み込む。
